In Display.ino, add the power case to the switch statement inside renderOperationalDisplay() (around line 253). Insert after the DISPLAY_MODE_SPEED case and before DISPLAY_MODE_BAT:

```cpp
case DISPLAY_MODE_POWER:  displayShowTwoDigitOrDash(telemetry.foil_power != 0xFF ? (uint8_t)(telemetry.foil_power / 2) : 0xFF); break;
```

Conversion logic: `foil_power` is encoded as watts/50 by the Rx. Dividing by 2 gives hectowatts (watts/100). So foil_power=50 (2500W) displays as "25". foil_power=16 (800W) displays as "08". The displayShowTwoDigitOrDash function handles 0xFF (shows "--") but we guard explicitly because the division would corrupt the sentinel.

Maximum displayable: foil_power=255 (not 0xFF sentinel, max real value=254) -> 254/2=127 -> would overflow 2-digit display. But values above 198 (9900W) are beyond any eFoil motor, so this is safe in practice. If concerned about overflow, clamp: `min((uint8_t)(telemetry.foil_power / 2), (uint8_t)99)`. Use the clamped version for safety:

```cpp
case DISPLAY_MODE_POWER:  displayShowTwoDigitOrDash(telemetry.foil_power != 0xFF ? min((uint8_t)(telemetry.foil_power / 2), (uint8_t)99) : 0xFF); break;
```

Do NOT change the default case or any other existing cases.
